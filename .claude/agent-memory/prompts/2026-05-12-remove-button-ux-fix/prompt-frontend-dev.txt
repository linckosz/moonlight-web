## Tache : Correction UX — Bouton Remove a la meme position que Open

### Contexte

Dans `frontend/js/ui/HostListView.js`, la methode `renderCard()` (lignes 267-274) affiche un bouton dans `.host-card-actions` selon l'etat du host :

```js
<div class="host-card-actions">
    ${host.isAvailable
        ? `<button class="btn btn-open" data-uuid="${host.uuid}">Open</button>`
        : host.isLocked
            ? `<button class="btn btn-secondary btn-pair" data-uuid="${host.uuid}">Pair</button>`
            : `<button class="btn btn-secondary btn-remove" data-uuid="${host.uuid}">Remove</button>`
    }
</div>
```

**Probleme :** Open et Remove sont exactement a la meme position dans le layout. Si un host devient indisponible entre le moment ou l'utilisateur vise "Open" et le clic, Remove apparait subitement a la meme place, et le clic supprime le host au lieu de l'ouvrir.

### Solution demandee

**Deplacer le bouton Remove dans un conteneur separe, place a un endroit different du layout.** Le Remove ne doit jamais partager la position du bouton Open.

Approche concrete :

1. **Structure HTML :** Dans `renderCard()`, ajoute un deuxieme conteneur `.host-card-remove` a cote de `.host-card-actions`. Le bouton Remove va dans `.host-card-remove` (toujours visible quand le host est offline), tandis que `.host-card-actions` ne contient que Open ou Pair (ou rien si offline).

   Nouveau layout de la carte :
   ```
   [icon] [info...] [status-badge] [actions: Open/Pair] [remove: x]
   ```

2. **Style CSS :** Dans `frontend/css/style.css`, ajoute les regles pour `.host-card-remove`. Le Remove doit etre visuellement distinct :
   - Plus petit qu'un bouton normal
   - Icone seule ou texte reduit (ex: "x" ou "✖" ou "Remove" en petit)
   - Couleur differente (gris/rouge discret, pas le meme style que btn-open)
   - Positionne a droite de `.host-card-actions`, ou en bas a droite de la carte

3. **Comportement :** Quand Remove est clique, le comportement existant est conserve (desactiver le bouton, appeler BackendClient.removeHost, re-rendre la liste).

4. **Responsive :** Sur mobile (max-width: 600px), le Remove doit rester accessible et ne pas chevaucher les autres elements.

### Fichiers a modifier

- `frontend/js/ui/HostListView.js` — restructurer renderCard()
- `frontend/css/style.css` — ajouter les styles pour `.host-card-remove`

### Contraintes

- Ne pas changer la structure globale de la carte (host-card reste un flex-row)
- Ne pas casser le systeme de fingerprint existant (ligne 162-168)
- Ne pas casser le delegation d'evenements (clickHandler ligne 28-73, qui utilise event delegation sur .btn-remove)
- Le Remove ne doit **jamais** etre a la meme position que le bouton Open
