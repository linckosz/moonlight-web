## Resume — Code changes only (build not run)

Les modifications suivantes ont ete apportees aux fichiers source :

### 1. `backend/src/server/AppSettings.cpp` — `domain()` always computes from uniqueId

- L'ancienne logique lisait d'abord le champ "domain" stocke dans settings.json et le retournait s'il etait valide.
- La nouvelle logique ignore le champ stocke et construit toujours le domaine depuis `uniqueId + baseDomain`.
- Le champ "domain" dans settings.json devient un cache write-only (ecrit par InternetAccessManager::ensureIdentifiers() mais plus lu par domain()).

### 2. `backend/src/main.cpp` — Startup cert domain sync + handler fix

- Nouveau bloc apres InternetAccessManager construction : verifie si MW_CERT_PEM/MW_CERT_KEY (env vars ou DEFINES) ont un CN qui correspond au domaine calcule.
- Si le CN correspond : ecrit `cert_pem="MW_CERT_PEM"` et `cert_key="MW_CERT_KEY"` dans settings.json ET sur le HttpServer, annulant d'eventuels chemins de fichiers LE leftover.
- Si le CN ne correspond pas : laisse settings.json tel quel, loadCert() tombera dans le fallback file scan ou ACME.
- Handler `certificateChanged` : ajout de `server.setDomain(appSettings.domain())` avant reloadTls(), pour que le domaine a jour soit utilise lors du reload.
- Ajout de l'include `#include <QSslCertificate>` pour le nouveau code de verification CN.
