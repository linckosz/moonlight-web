## Contexte

Le stream échoue avec le codec HEVC. Le flow est :

1. Un keyframe HEVC arrive dans le navigateur
2. `configureDecoder()` est appelé — `isConfigSupported` retourne false pour `hvc1.4.0.L0.B0` mais true pour `hvc1.1.6.L153.B0`
3. `decoder.configure()` réussit (state=configured)
4. `dequeuePendingFrames()` est appelé immédiatement après
5. `decode()` échoue car le decoder est déjà "closed" — une fermeture silencieuse asynchrone

C'est une race condition WebCodecs : configure réussit synchronement, mais le decoder peut se fermer asynchronement (erreur matérielle, pilote, etc.).
